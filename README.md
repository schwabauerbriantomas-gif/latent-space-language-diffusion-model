# SplatDiffusion

A masked diffusion language model (MDLM) for text generation, built on top of SplatsDB's vector infrastructure. This repo explores whether masked diffusion — which generates tokens in parallel rather than autoregressively — can achieve competitive throughput while maintaining text quality.

## Results at a Glance

| Metric | Value | Notes |
|--------|-------|-------|
| Model parameters | 201M | d_model=1024, 10 layers, 16 heads |
| Training data | 272M tokens | 1M docs from Ultra-FineWeb |
| Perplexity (held-out) | 102.6 | Qwen3-0.6B reference: ~15-20 |
| Forward throughput | 57,759 TPS | Batch=32, vs Qwen3's 32,023 (1.8× faster) |
| Generation speed | 52 tok/s | Semi-AR with adaptive guidance |
| Repetition score | 0.99 | After logit guidance (baseline: 0.79) |

**Hardware**: Single NVIDIA RTX 3090 (24GB VRAM), 8GB system RAM.

## Architecture

```
MDLM-BPE v3 (201M params)
├── Token embedding: 16K BPE vocab → 1024 dims
├── 10× Transformer blocks
│   ├── RoPE positional encoding
│   ├── AdaLN timestep conditioning
│   └── Flash attention (non-causal)
├── LayerNorm + output projection
└── Semi-autoregressive unmasking (left-to-right blocks of 4)
```

**Key design choice**: Instead of predicting all positions simultaneously (fully parallel), the model generates text in left-to-right blocks of 4 tokens. Each block has full context of everything to its left, while tokens within a block are predicted in parallel via diffusion. This trades some speed for significantly better coherence.

### Generation Pipeline

```
Prompt → [Block 1: 4 tokens in parallel] → commit
       → [Block 2: 4 tokens with block 1 as context] → commit
       → ... until sequence complete

Each block uses:
  1. Adaptive temperature cooling (1.0 → 0.5 across blocks)
  2. Top-p nucleus sampling (p=0.95)
  3. Adaptive repetition penalty (1.2 → 1.5)
  4. Adaptive frequency penalty (0.3 → 0.6)
  5. No-repeat bigram ban
```

### HRM (Hierarchical Refinement Model)

Optional post-generation refinement with two layers:

1. **RepetitionReviewer** (0 params, geometric rules): Detects exact repeats, window repetition, and bigram duplication. Masks bad positions for regeneration.

2. **AR-Oracle** (Qwen3-0.6B): Scores generated text via teacher forcing. Low-probability segments are regenerated by the oracle. See `src/hybrid_speculative.py`.

## Benchmark [MEASURED]

All metrics measured on RTX 3090 with identical prompts. Oracle log-prob = mean per-token log-probability under Qwen3-0.6B teacher forcing (higher = more coherent).

| Method | Oracle LP | Rep Score | TPS | Latency |
|--------|:---------:|:---------:|:---:|:-------:|
| MDLM baseline | -3.38 | 0.84 | 51.8 | 1.1s |
| MDLM + guidance | -4.29 | 0.98 | 52.2 | 1.1s |
| MDLM + guidance + RepHRM | -4.35 | 0.99 | 51.0 | 1.1s |
| Hybrid (MDLM draft + Qwen3 refine) | -3.99 | 0.99 | 22.9 | 1.9s |
| Qwen3-0.6B (pure AR) | -1.18 | 0.95 | 17.1 | 3.8s |

**Forward-pass throughput** (no generation loop, just model inference):

| Batch | MDLM | Qwen3-0.6B | Speedup |
|------:|-----:|-----------:|--------:|
| 1 | 8,443 TPS | 2,152 TPS | 3.9× |
| 8 | 50,440 TPS | 17,009 TPS | 3.0× |
| 32 | 57,759 TPS | 32,023 TPS | 1.8× |

The MDLM produces grammatically correct, repetition-free text. Semantic coherence is limited by model scale (see Limitations below).

## Honest Limitations

This project demonstrates that masked diffusion is a viable architecture for fast text generation. However, the **quality gap** between the MDLM and a production AR model is significant:

| | MDLM v3 | Qwen3-0.6B | Ratio |
|--|--------:|-----------:|------:|
| Parameters | 201M | 596M | 0.34× |
| Training tokens | 272M | ~trillions | ~0.0003× |
| Oracle log-prob | -3.38 | -1.18 | — |
| Perplexity | 102.6 | ~15-20 | ~5× worse |

**The quality gap is driven by scale, not architecture.** The MDLM was trained on 272M tokens (1M documents) on a single RTX 3090 in ~7 hours. Production models train on trillions of tokens across GPU clusters. The throughput advantage of masked diffusion (1.8-3.9× faster forward pass) is real and architecture-level; the coherence gap is a resource constraint.

### What Guidance Can and Cannot Fix

**Logit guidance eliminates repetition** (0.79 → 0.99 repetition score) at zero computational cost. This is the clear win.

**Guidance cannot fix semantic coherence.** Three HRM approaches were tested:

| Approach | Δ log-prob | Verdict |
|----------|:----------:|---------|
| Semantic HRM (MDLM embeddings) | — | Failed: embeddings too weak |
| AR-Oracle guided regeneration | -0.02 | Failed: MDLM too weak to generate good replacements |
| AR-Oracle token replacement | -0.71 | Failed: cross-vocabulary mapping breaks coherence |
| Hybrid segment regeneration | +0.22 | Partial: 4/6 prompts improved |

Full experiment details in [`docs/phase10_results.md`](docs/phase10_results.md).

## Reproduce

### Train from scratch

```bash
# 1. Train BPE tokenizer (16K vocab)
python scripts/train_tokenizer.py

# 2. Download and prepare data
python scripts/download_data.py          # Ultra-FineWeb, 1M docs
python scripts/prepare_data.py           # Tokenize → memmap

# 3. Train MDLM v3 (~7 hours on RTX 3090)
python scripts/train.py --epochs 3 --batch-size 32 --seq-len 128

# 4. Benchmark
python scripts/benchmark.py
```

### Requirements

- Python 3.10+
- PyTorch 2.6+ with CUDA
- NVIDIA GPU (tested on RTX 3090, 24GB VRAM)
- transformers (for Qwen3 oracle in hybrid mode)
- tokenizers (HuggingFace BPE)

```bash
pip install torch transformers tokenizers numpy
```

## Research Lineage

This repo documents a complete research arc. The `experiments/` directory preserves the full history:

| Phase | Question | Outcome |
|-------|----------|---------|
| 1-4 | Can FF/score-matching generate text in SplatsDB latent space? | ❌ Energy collapse in 1024D |
| 5 | Can masked diffusion generate grammatical text? | ✅ 100% grammatical (small vocab) |
| 6 | Can HRM (Generator→Reviewer→Editor) improve quality? | ✅ 84% grammatical, 635 vocab |
| 7 | Can SplatsDB latent space control topic? | ✅ 79% on-topic, 10.3× over baseline |
| 8 | Can the system detect and fill knowledge gaps? | ✅ 100% gap detection accuracy |
| 9-10 | Can MDLM scale to coherent real text? | 🔶 Fast (1.8-3.9×) but quality-limited by scale |
